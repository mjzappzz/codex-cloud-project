总流程

1. 选择系统

根据 Rocky 9.4 或 Ubuntu 22.04 进入对应安装分支。

2. 安装软件包

下载对应的 RPM 或 DEB 安装包，执行安装命令。

3. 补齐依赖

安装桌面、图形、音频与 Qt 相关依赖库。

4. 排障与验证

处理黑屏、检查缺失库，并按需重启服务。

黑屏处理

ToDesk 常见黑屏原因之一是桌面环境使用了 Wayland。可以改为 Xorg 并重启显示管理器与 ToDesk 服务。

1. 修改 GDM

sudo sed -i 's/^#WaylandEnable=.*/WaylandEnable=false/' /etc/gdm/custom.conf

2. 修改 GDM3

sudo sed -i 's/^#WaylandEnable=.*/WaylandEnable=false/' /etc/gdm3/custom.conf

3. 重启桌面

sudo systemctl restart gdm sudo systemctl restart gdm3

4. 重启 ToDesk

sudo systemctl restart todeskd.service

依赖排查

如果安装后无法启动，优先检查运行时库是否缺失。

ldd /opt/todesk/bin/ToDesk_Service | grep "not found"

输出中若存在 `not found`，继续安装对应系统缺失的库。

补充操作

Rocky 卸载 sudo yum remove -y todesk
Ubuntu 卸载 sudo apt remove -y todesk
使用建议
如果是服务器或远程桌面环境，优先确认当前登录会话是否使用 Xorg，避免把安装问题和图形会话问题混在一起。
